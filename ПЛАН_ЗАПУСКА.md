## 0. Что это за проект (коротко)

Telegram **mini-app** (приложение внутри Telegram) на **Next.js 14 + Supabase**.
Пользователь заходит из Telegram → видит каталог грибных добавок → оформляет заказ → копит монеты (SC) → приглашает друзей (рефералка).

**Стек:**
- Frontend + API: Next.js 14 (папка `app/`)
- База данных: Supabase (PostgreSQL)
- Telegram-бот: `tg-bot/` (отдельный процесс на Node.js)
- Деплой (планируется): из GitHub → Vercel + домен; позже возможен VPS

## 🎓 Модуль обучения «Академия микродозинга» (после запуска)

Серия экранов про микродозинг: пользователь проходит разделы, видит прогресс и получает токены (SC).

**Решения (зафиксировано 2026-06-23):**
- Зачёт урока — **мини-квиз** (1–2 вопроса в конце). Защищает от фарма токенов.
- Порядок — **свободный доступ** к любому уроку, прогресс просто отмечается.
- Сроки — делаем **после базового запуска** (этапы 0–5).

**Три типа экранов:**
1. Обзор курса — карточки разделов (пройдено ✓ / доступно / 🔒), прогресс-бар, счётчик полученных/доступных SC.
2. Экран урока — текст + карточки-блоки; тап по карточке → всплывающее окно с описанием и кнопкой «▶ Смотреть видео».
3. Квиз — 1–2 вопроса; верный ответ → начисление токенов (один раз) и отметка «пройдено».

**Переиспользуем существующее:**
- Начисление токенов — таблица `coin_transactions`, паттерн как в `api/subscribe-bonus` (идемпотентно, без двойного начисления). `type = 'lesson_<slug>'`.
- Хранение контента — `lib/contentManager.ts` + `api/content-management` (уроки редактируются без правки кода).
- Навигация — новая вкладка в `AppClient.tsx`.
- Геймификация — токены за обучение автоматически двигают уровни (уровни считаются по сумме SC).

**Новые таблицы Supabase:**
- `lessons` — id, slug, title, order_index, description, reward_sc, content_blocks (jsonb: карточки {заголовок, текст, video_url}), video_url, quiz (jsonb), is_published.
- `lesson_progress` — user_id, lesson_id, status (available/completed), completed_at.
